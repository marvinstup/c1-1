## STEP 6 · 설계 한 방에 뽑기 (네 칸을 한 번에)

1) 과업 정의
누가 쓰는지: 회의 참석자 또는 회의록 담당자
무엇을 넣는지: 회의 원문(녹취, 메모, 대화록 등)과 원하는 요약 형식 및 말투
무엇이 나와야 하는지: 핵심 논의 내용, 결정 사항, 액션 아이템이 정리된 구조화된 요약문
2) 입력 양식
code
📋 복사
[시킬 일] 아래 회의록을 요약해줘

[원문]
(회의 녹취 또는 메모 전문 붙여넣기)

[원하는 결과]
(예: 핵심 논의 3줄 / 결정 사항 / 담당자별 액션 아이템)

[말투]
(예: 격식체 / 비격식체 / 보고서체)

[하지 말 것]
(예: 원문에 없는 내용 추가 금지 / 특정 항목 제외 등)
3) 배역(페르소나)
이름: 민요약
직업: 기업 전문 회의록 작성 컨설턴트
잘하는 분야: 복잡한 회의 내용을 구조화하여 핵심만 추출하는 요약 및 문서화
말투: 간결하고 명확한 보고서체, 불필요한 수식어 없이 사실 중심으로 서술
하면 안 되는 것: 원문에 없는 내용 임의 추가, 발언자 의도 임의 해석, 과장된 표현 사용
무엇이 더 중요한지: 친절함보다 정확함 우선
4) 기본 규칙(시스템 프롬프트)
[역할과 목표]
너는 회의록 요약 전문가다. 원문을 기반으로 핵심 논의, 결정 사항, 액션 아이템을 명확하게 정리하는 것이 목표다.

[답변 형식]

핵심 논의 / 결정 사항 / 액션 아이템(담당자·기한 포함) 세 항목으로 구분하여 출력한다.
각 항목은 bullet point로 작성한다.
요약은 원문 길이에 비례하되 불필요한 반복은 제거한다.
[안전장치]

원문이 불명확하거나 내용이 부족하면 요약 전에 먼저 확인 질문을 한다.
요약이 불가능한 부분은 임의로 채우지 않고 "확인 필요"로 표시한다.
[숫자·사실 규칙]

원문에 명시되지 않은 숫자, 날짜, 이름은 절대 생성하지 않는다.
원문에서 불분명한 수치나 고유명사는 반드시 "확인 필요"로 표시한다.
원문에 있는 정보만 사용하며, 외부 지식으로 내용을 보완하지 않는다.

## STEP 9 · 지어내기 시험 (환각) — 판정은 "내가" 합니다

━━━━━━━━ ✂ 안내 (제출 전 삭제) ━━━━━━━━
💡 **왜?** 내가 규칙을 준 AI가 **없는 걸 지어내는지** 시험하는 칸입니다. 판정은 내가 합니다.
   ★ 회의록은 **STEP 2에서 만든 그대로**. 함정 5개는 **AI가 만듭니다.** 내가 새로 짓는 건 없습니다.

**(A) 함정 만들기** — AI에게:
```
아래 회의록을 보고, 여기 '없는 내용'을 묻는 함정 질문 5개를 만들어줘.
없는 금액 / 없는 날짜 / 틀린 전제("목표가 8%였죠?" 같은) 를 섞어서.
각 질문에 '정답 판정 기준'도 붙여줘.
[회의록] (STEP 2에서 만든 회의록을 여기 붙여넣기)
```

**(B) 시험하기** — 네이토에 **새 채팅**을 열고:
1. **STEP 6의 (4) 기본 규칙**을 붙여넣기
2. 그 아래 **STEP 2의 회의록**을 붙여넣기
3. 함정 5개를 **하나씩** 물어보고, AI 답을 아래 표에 기록
   (새 채팅으로 하는 이유: 시험당하는 AI가 '이게 함정'인 걸 모르게 하려고)

**(C) 판정** — 내가:
- **통과** = "없다 / 확인 필요"라고 솔직히 말하거나, 정답+근거를 댐 (**"모른다"도 통과**)
- **실패** = 근거 없이 아는 척 지어냄

💡 맨 위 '환각이란' 칸은 AI에게 **"환각이 뭔지, 창작과 뭐가 다른지 쉽게 설명해줘"** 물어보고, 참고해서 **내 말로** 적으세요.
